# 小口売り連続検知ストラテジー（検知・通知のみ）

買い気配に対して小口の売りが連続してぶつかり、かつ価格が崩れない状態を検知し、
需給改善の兆し（＝上抜けサインの可能性）として通知するツールです。

**現段階では自動発注は行いません。** シグナルの検知結果をログとポップアップ通知に出すところまでです。
発注ロジックを組み込むかどうかは、この検知ロジックの実運用結果を見てから改めて相談してください。

## 重要な前提（必ず読んでください）

kabuステーションAPIのPUSH配信には、**1約定ごとの約定株数（歩み値）を返す専用フィールドは存在しません。**
[公式GitHub Issue #268](https://github.com/kabucom/kabusapi/issues/268) でも同様の要望が出ていますが、
現行ドキュメントに実装された記載はありません。

そのため本ツールでは、PUSH配信に含まれる **当日累計出来高（`TradingVolume`）の差分** を
「直近の推定約定株数」として扱う近似ロジックを採用しています。

- 複数の約定がほぼ同時に発生すると、1回のPUSH更新に出来高がまとめて反映されることがあります。
  この場合「小口が5回連続」ではなく「小口相当のPUSHが5回連続」という近似になります。
- `debug_raw_messages` の設定で、起動直後のPUSH生データをそのままログに出せます。
  本番運用前に、実際のJSONの `Symbol` / `CurrentPrice` / `TradingVolume` / `Buy1` の
  値が想定通り動いているか、必ず目視で確認してください。
- **フィールド名の注意**: kabuステーションAPIは `BidPrice`=最良「売」気配 /
  `AskPrice`=最良「買」気配という、一般的な英語の慣例と**逆**の命名です。
  本ツールでは誤解の余地がない `Buy1.Price`（買い気配1本目）を最良買い気配として使用しています。

## シグナルの判定仕様

「直前のPUSH時点の最良買い気配」と「新しく発生した約定の価格」を比較して判定します。

- **カウントする約定（ヒット）**: 出来高が増加し、約定価格が直前の買い気配以下
  （＝売り方起点。買い気配を食い破って下の価格で約定したケースも含む）で、
  かつ推定約定株数が `small_lot_threshold` 以下。
- **リセットされる約定**: 大口の約定、または売り気配を買い上がる約定（買い方起点）。
- **買い気配の価格が動いてもストリークは継続**します。「その時々の買い気配に小口売りが
  ぶつけられ続けているか」を数える仕様です（価格下落による制限は設けない）。
- 約定を伴わない板だけの更新は、カウントもリセットもしません。
- **アルゴ分散売り対策（時間間隔フィルタ）**: 直前の小口売りから
  `min_hit_interval_seconds`（既定1秒）以上離れた小口売りだけをカウントします。
  1秒未満の間隔で連射される小口売りは「同一の大口注文を分割したもの」とみなし、
  **カウントもリセットもしません**。連射が続いている間はカウントが進まず、
  売りが1秒以上途切れてから来た次の小口売りで初めて次のカウントが入ります
  （急落時のアルゴ連射を「小口売り5回」と誤認しないため）。
  時刻はPUSHデータの `TradingVolumeTime`（売買高時刻）を使用します。
  大口の約定・買い方起点の約定は、間隔にかかわらず従来どおりリセットします。

※「価格が下がってもカウントを続ける」仕様のため、小口売りに押されて下落し続けている
弱い相場でもシグナルが発火し得ます。運用ログを見て、必要なら下落制限の追加を検討してください。

## セットアップ

1. kabuステーション（デスクトップアプリ）を起動し、**検証（デモ）環境**にログインしておく。
2. kabuステーションのAPI設定でAPIパスワードを確認し、環境変数に設定する。

   ```
   set KABU_API_PASSWORD=あなたのAPIパスワード
   ```

3. 依存パッケージをインストールする。

   ```
   pip install -r requirements.txt
   ```

4. `config.example.yaml` を `config.yaml` にコピーし、監視銘柄・閾値を編集する。

## 実行

```
cd src
python main.py --config ../config.yaml
```

- `environment: demo` の場合はポート18081（検証環境）、`production` の場合はポート18080（本番環境）に接続します。
- 起動時に既存のPUSH登録銘柄をすべて解除してから、`config.yaml` の銘柄を登録し直します。
- 終了する場合は `Ctrl + C` してください。

## パラメータの意味（config.yaml）

| キー | 意味 |
|---|---|
| `environment` | `demo`（検証環境） または `production`（本番環境） |
| `symbols` | 監視銘柄のリスト（最大50銘柄、kabuステーションAPIのPUSH配信の上限） |
| `small_lot_threshold` | この株数（推定約定株数）以下を「小口」とみなす |
| `price_tolerance_ticks` | 最良買い気配との価格差の許容ティック数（0=完全一致のみをヒットとみなす） |
| `alert_tiers` | 連続ヒット回数ごとの通知段階（例: 5回でWATCH、10回でSTRONG） |
| `debug_raw_messages` | 起動直後、指定件数分のPUSH生データをログにそのまま出力する |

## 今後の拡張について

自動発注（成行・指値での買い注文送信）を追加する場合は、次の点を必ず事前に相談・確認してから実装します（[プロジェクト方針](../../CLAUDE.md)より）。

- ポジションサイズ（1回あたりの発注株数・金額）
- 損切り条件（％、または金額）
- 日次の最大損失額・最大同時保有銘柄数
- 本番環境に切り替えるタイミングと判断基準
