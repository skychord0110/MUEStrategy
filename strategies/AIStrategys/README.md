# AIStrategys — ログ分析に基づくAI提案ストラテジー（仮想売買）

2026-07-13〜07-17のrunnerログ分析（[../../analysis/](../../analysis/README.md)）から
導いた買い戦略を、**発注なしの仮想売買（ペーパートレード）**として実装したもの。
統合ランナー（strategies/runner）に組み込まれており、`runner/src/main.py` を
実行すれば他のストラテジーと同様に検知・仮想売買の結果が `runner/logs/` に記録される。

## 収録ストラテジー

### 1. afternoon_reversal — 午後の下値大口買い検知・引け戻り

- **エントリー**: 13:00〜15:00にUNDER急増アラートが点灯し、**現在値が500円以上**の銘柄を、
  その時点の現在値で仮想買い（同一銘柄は1日1回まで）
- **決済**: 損切り-2% / 利確+2%、どちらにも達しなければ大引け（15:30）
- **根拠（2026-07-13〜17の初期検証）**: 23トレード・勝率69.6%・期待値+0.63%/回
  （スリッページ1ティック控除後）。前場のシグナルは期待値-1%のため使わない
- **価格下限フィルタ（min_entry_price=500）の追加根拠（2026-07-13〜24の両週検証）**:
  500円未満の低位株は勝率50%とノイズが大きく、500円以上に絞ると勝率71%→81%・
  期待値+0.79%/回に改善（両週ともプラスで頑健）。`analysis/analysis_result_2026-07-24.md` 参照。
  `min_entry_price: 0` にすればフィルタ無効（旧挙動）
- **今週(07-21〜24)の実稼働フォワード実績（フィルタ導入前）**: 12トレード・勝率63.6%・
  期待値+0.115%/回。バックテストより劣化しており、フィルタ版も実運用では下振れる前提で見ること

### 2. confluence — 複合シグナル

- **エントリー**: 13:00〜15:00に、30分以内で「**UNDER急増**」と「**小口売り連続**」の
  **両方**が同一銘柄に点灯したら、その時点の価格で仮想買い（同一銘柄は1日1回まで）。
  分析で有効だったのはこの2種の組み合わせのため、投げ売り検知は判定に使わない
- **決済**: 損切り-1% / 利確+1%、どちらにも達しなければ大引け
- **根拠（同上）**: 勝率80%・期待値+0.65%/回。**ただしn=5と極小サンプル**のため、
  まずはこの仮想売買ログでフォワード検証する位置づけ

## ログの見方

```
[AI午後引け戻り/エントリー] 4165 4165: UNDER急増を検知、560.0円で仮想買い（損切り-2.0%/利確+2.0%/残りは大引け・発注なし）
[AI午後引け戻り/決済:利確] 4165 4165: 仮想決済 560.0円→571.5円 (+2.05%)
[AI複合シグナル/エントリー] 6620 6620: small_lot_sell_detector+under_surge_detector が30分以内に点灯、735.0円で仮想買い（…）
[AI複合シグナル/決済:大引け] 6620 6620: 仮想決済 735.0円→741.0円 (+0.82%)
```

決済理由は `利確` / `損切り` / `大引け` / `大引け(補完)` の4種。
「大引け(補完)」は当日中に大引けのPUSHが受信できなかった銘柄を、
翌営業日の最初のメッセージ時に当日最後の観測値で決済扱いにしたもの。

週次の勝率・期待値の集計は、ログの `決済:` 行の `return_pct` を集めれば算出できる
（analysis/ のスクリプトと同じ要領）。

## 設定（runner/config.yaml の strategies セクション）

```yaml
  afternoon_reversal:
    enabled: true
    entry_start: "13:00"   # 仮想エントリーを許可する開始時刻
    entry_end: "15:00"     # これ以降は新規エントリーしない
    stop_loss_pct: 2.0
    take_profit_pct: 2.0
    min_entry_price: 500.0 # この価格未満の銘柄はエントリーしない（0で無効）

  confluence:
    enabled: true
    window_seconds: 1800   # UNDER急増と小口売り連続がこの秒数以内に両方揃ったら発火
    entry_start: "13:00"
    entry_end: "15:00"
    stop_loss_pct: 1.0
    take_profit_pct: 1.0
```

confluenceは基礎ストラテジー（under_surge_detector / small_lot_sell_detector）の
アラートを入力とするため、この2つのどちらかを `enabled: false` にすると発火しなくなる。

## 重要な注意

- **このストラテジーは発注を一切行わない。** 実発注に進む場合は、
  仮想売買ログで2〜4週間フォワード検証し、勝率・期待値が分析時の水準を
  維持しているかを確認したうえで、別途実装・レビューすること（CLAUDE.mdの方針）
- 検証サンプルは1週間分で、利益の大半が2026-07-17に集中している（地合い依存の可能性）
- 仮想決済の損切り/利確は現在値ベースの判定であり、実際の約定価格は
  板の厚さ・スリッページによりこれより不利になり得る
