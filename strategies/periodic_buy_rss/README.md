# periodic_buy_rss — マーケットスピードII RSS（歩み値）による定期買い集め検知

楽天証券マーケットスピードII RSS の `RssTickList`（歩み値）を使い、**ある約定（多くは売り）の
丁度10秒後に買いが入る**動きが1日に何度も（既定5回以上）繰り返されているときに通知する。
kabuステーション版（`strategies/periodic_buy_detector`）と検知の狙いは同じだが、入力が
**1約定ごとの正確な時刻・株数（歩み値）**になるため、当日累計出来高の差分による近似より精度が高い。

検知・通知のみで**発注は行わない**。データ源が楽天証券なのでkabuランナーとは別プロセスで動く
独立ツール（プロジェクトの発注はkabuステーションAPIのまま。本ツールは分析・検知専用）。

## 仕組み

1. 起動中のExcel（RSSアドイン有効・マーケットスピードIIログイン済み）に win32com でアタッチ
2. 各銘柄について `=RssTickList(,"<コード>.T",300)` をシートに書き込む
   → 直近300件の歩み値（**時刻・出来高・約定値**）がスピル表示される
3. 一定間隔でスピル範囲をポーリングし、前回との差分（新規約定）だけを抽出
4. **売買方向はティックルールで推定**（約定値が直前より高い→買い / 低い→売り / 同値→直前を継承）
   ※歩み値には売買の別が無いための近似
5. 「売り約定の丁度N秒後の買い」を当日カウントし、5回でWATCH・10回でSTRONG通知

ログは `periodic_buy_rss/logs/periodic_buy_rss_YYYY-MM-DD.log` に日付別で保存。

## セットアップ

### 前提（ユーザー側で用意）

1. **マーケットスピードII** を起動しログインしておく（RSSは起動中のみ更新される）
2. **Excel に「マーケットスピードII RSS」アドインを有効化**しておく
   （楽天証券のRSS設定手順に従う。Excelは通常のデスクトップ版・Windows）
3. **空のExcelブックを1つ開いておく**（本ツールがそこに `TICKS` シートを作り数式を書き込む）

### インストールと実行

```powershell
cd strategies\periodic_buy_rss
pip install -r requirements.txt
# config.yaml は作成済み（config.example.yamlと同内容）。閾値や銘柄設定を変える場合は編集
cd src
python main.py --config ../config.yaml
```

停止は `Ctrl + C`。監視銘柄は既定で全ストラテジー共通の `strategies/symbols.yaml` を使う。

## 設定（config.yaml）

- `rss.*`: Excel連携（市場記号・取得本数・シート名・ポーリング間隔など）
- `detector.*`: 検知条件（kabu版と同じ意味。`delay_seconds`/`trigger_side`/`alert_tiers` 等）
- `debug_raw_batches`: 起動直後N回、読み取った歩み値の直近3件をログ出力（項目・時刻形式の確認用）

## ログの見方

```
[アルゴ買い集め(RSS)/WATCH] 4165 4165: 売り約定の約10.0秒後に買い上がる動きを本日5回検知
（歩み値ベース・現在値560円）。一定ラグで反応する買い集めアルゴの可能性（大量保有報告に向けた仕込みサインの疑い）
```

## 重要な前提・限界

- **売買の別は歩み値に含まれず、ティックルールで推定**している。気配（板）と突き合わせる方式より
  精度は落ちる。特に同値約定が続く場面では方向の取りこぼしがある
- `RssTickList` は「直近最大300件」を返す上書き表示のため、**ポーリング間隔が空きすぎて300件を
  超えて約定が進むと取りこぼす**。活況銘柄では `poll_interval_seconds` を短く/ `tick_count` を大きく
- **「本日N回」は正確には「ツール起動後に観測した約定の中でのN回」**。RssTickListは起動時点で
  直近300件しか遡れないため、1日を通した正確なカウントには**ザラ場開始前（9:00前）から
  起動し続ける**こと。引け後に起動した場合は直近300件（＝終盤の数分）だけが対象になり、
  大引け(15:30)の同時約定が混じるため参考値程度に見ること
- 歩み値の**時刻は秒単位**（環境により表示形式が異なる）。`delay_tolerance_seconds` は1秒前後を推奨。
  初回は `debug_raw_batches` の出力で実際の時刻形式・並び順（newest_first）を必ず確認すること
- win32com を使うため **Windows + デスクトップExcel 専用**。ロジック自体は
  `tick_detector.py`（純Python）に分離してあり、Excel非依存で単体テスト可能

## トラブルシューティング

- **`呼び出し先が呼び出しを拒否しました` / `RPC_E_CALL_REJECTED (-2147418111)`**:
  Excelがビジー（セル編集中・ダイアログ表示中・RSS再計算中など）でCOM呼び出しを
  受け付けられない状態。本ツールは自動でリトライする（`com_retries`×`com_retry_delay`）。
  それでも出続ける場合は、Excelでセル編集を確定（Escや別セルクリック）し、開いている
  ダイアログを閉じてから再実行する。`config.yaml` の `rss.com_retries` を増やすのも有効
- **`起動中のExcelに接続できませんでした`**: Excelが起動していない。Excelを開いてから実行する
- 数式は書けたが値が空/エラー: マーケットスピードII本体が未起動・未ログイン、または
  RSSアドインが無効。`[DEBUG BATCH]` に何も出ない場合はまずこれを疑う

## 実装メモ（要現地確認）

この環境ではExcel/マーケットスピードIIを起動できないため、Excel連携部（`ms2_rss.py`）は
公式ドキュメントの `RssTickList` 仕様に基づく実装で、**実機での初回動作確認が必要**。
初回起動時の `[DEBUG BATCH]` ログ（時刻の形式・列の並び・新しい順か古い順か）を見て、
必要なら `rss.newest_first` や `parse_trade_time` の対応形式を調整する。
