# analysis — 場中ログの分析スクリプト

runnerのログ（検知アラート）を買いシグナルとみなした場合の勝率・期待値を検証する
分析パイプライン。ファイル名末尾の日付は分析を実施した日（対象データはコード内に記載）。

## 2026-07-18 実施分（対象: 2026-07-13〜07-17のrunnerログ）

実行順（上から順に。出力は `output/` に生成される）:

| # | スクリプト | 内容 | 出力 |
|---|---|---|---|
| 1 | `parse_alerts_2026-07-18.py` | runnerログから全アラート（681件）を抽出 | `output/alerts.csv` |
| 2 | `fetch_5m_2026-07-18.py` | 対象銘柄の5分足をYahoo Financeから取得 | `output/bars_5m.json` |
| 3 | `evaluate_2026-07-18.py` | 各アラートを買いエントリーとして決済ルール別に検証 | `output/eval_results.json`＋集計表 |
| 4 | `drilldown_2026-07-18.py` | 午後シグナル・日別頑健性・複合シグナルの深掘り | コンソール出力 |
| 5 | `final_sim_2026-07-18.py` | 実運用想定（銘柄1日1回・コスト控除）シミュレーション | コンソール出力 |

実行例（Windows / 文字化け防止に `-X utf8` を付ける）:

```powershell
cd analysis
python -X utf8 parse_alerts_2026-07-18.py
python -X utf8 fetch_5m_2026-07-18.py
python -X utf8 evaluate_2026-07-18.py
python -X utf8 drilldown_2026-07-18.py
python -X utf8 final_sim_2026-07-18.py
```

## 2026-07-18 分析の主な結論

- アラートを時間帯を問わず無差別に買うと期待値マイナス（勝率34%・-0.60%/回）
- **13:00以降のシグナルのみ期待値プラス**。前場（特に9:30〜10:00）は期待値-1%で買い禁止ゾーン
- 最良: **午後のUNDER急増→SL-2%/TP+2%/残りは大引け決済**
  → 23トレード・勝率69.6%・期待値+0.63%/回（スリッページ1ティック控除後）
- 午後に30分以内で「UNDER急増＋小口売り連続」が両方点灯した複合シグナルは勝率80%・+0.65%（ただしn=5）
- 小口売り連続は単独の買いシグナルとしては期待値マイナス

この結論をもとに `strategies/AIStrategys/` の2ストラテジー
（afternoon_reversal / confluence）を実装している。

## 注意事項

- Yahoo Financeの5分足は**直近約60日分しか取得できない**。対象週が古くなると
  再取得不能になるため、`output/bars_5m.json` は消さずに保管すること
- 検証は5分足ベースの近似。損切りはストップ価格で約定した前提（ギャップ滑りは未考慮）、
  同一足でSL/TP両到達の場合は損切り優先の保守的処理
- サンプルは1週間分のみ。利益の大半が2026-07-17に集中しており地合い依存の可能性が高い。
  結論の採用前にフォワード検証（AIStrategysの仮想売買ログ）を重ねること

## 次回以降の分析

新しい週を分析する場合は、スクリプトをコピーして日付サフィックスを更新し、
対象ログのglobパターン（parse）と取得期間のepoch（fetch）を書き換える。
